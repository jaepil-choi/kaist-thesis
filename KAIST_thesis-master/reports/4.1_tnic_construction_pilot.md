# Phase 4.1: TNIC Construction with Median Adjustment (Pilot: 2010-2011)

**Generated**: 2025-10-29

## Overview

This phase implements Text-based Network Industry Classification (TNIC) following Hoberg & Phillips (2016) methodology, applying it to Korean market data for years 2010-2011 as a pilot.

## Methodology

### Citation

**Hoberg, G., & Phillips, G. M. (2016).** Text-based network industries and endogenous product differentiation. *Journal of Political Economy*, 124(5), 1423-1465.

### TNIC Key Properties

**1. Firm-Specific Peer Groups (No Fixed Location)**

Unlike traditional industry classifications (SIC, FnGuide), TNIC allows each firm to have its own set of peers:
- Firm i's peers = firms with high text similarity to firm i
- Different firms can have completely different peer sets
- No requirement for "industry membership" - each firm defines its own competitive space

**2. Symmetric but Non-Transitive**

The raw similarity matrix is **symmetric**: `sim(i,j) = sim(j,i)`

Therefore, peer relationships based on **raw** scores are symmetric:
- If firm i considers j as a peer → firm j also considers i as a peer

**However, there is NO transitivity:**
- Firm A and B are peers: sim(A,B) > threshold ✓
- Firm B and C are peers: sim(B,C) > threshold ✓
- **But A and C might NOT be peers**: sim(A,C) < threshold ✗

This breaks the "clique" property of traditional industry classifications.

**3. Median Adjustment Creates Asymmetry**

After median adjustment (described below), the similarity matrix becomes **asymmetric**:
- `sim_adjusted(i,j) ≠ sim_adjusted(j,i)` in general
- This means peer relationships can become one-directional after adjustment

### Median Adjustment Procedure

**From H&P (2016), Section II.C:**

> "We consider one further refinement to further mitigate the impact of document length. For a firm i we compute its median score as the median similarity between firm i and all other firms in the economy in the given year. Intuitively, because no industry is large enough to span the entire economy, this quantity should be calibrated to be near zero. We achieve this by subtracting these median scores from the raw scores to obtain our final scores used for each firm."

**Implementation:**

For each firm i in year t:

```python
# Step 1: Compute firm-specific median
median_i = median(M_raw[i, :])  # median over all other firms

# Step 2: Subtract median from raw scores
M_adjusted[i, :] = M_raw[i, :] - median_i
```

**Why this matters:**
- Controls for document length effects
- Normalizes similarity scores relative to firm's "baseline" similarity
- Ensures that "no industry is large enough to span the entire economy"
- Median should be near zero after adjustment

### Threshold Calibration

**Objective:** Find threshold τ such that TNIC membership pairs fraction matches FnGuide Industry baseline.

**Two-Step Process:**

**Step 1: Calibrate on Raw Scores**
- Use raw similarity matrix M_raw
- Count pairs where M_raw[i,j] > τ
- Find τ where membership pairs fraction ≈ target (e.g., 2.95% for FnGuide Industry)

**Step 2: Define Peers Using Adjusted Scores**
- Apply median adjustment to get M_adjusted
- Define TNIC peers using calibrated threshold τ on M_adjusted
- Peers_i = {j : M_adjusted[i,j] > τ}

**Note:** We calibrate on raw scores for simplicity, but use adjusted scores for final peer definition following H&P (2016).

### Processing Steps

1. **Load Raw Similarity Matrices**
   - Load M_2010 (964×964), M_2011 (1,023×1,023)
   - Load firm mappings (stock_code ↔ matrix index)

2. **Match with FnGuide Data**
   - Filter FnGuide for EOY (December) data
   - Match stock codes between TNIC and FnGuide
   - Extract FnGuide Industry classifications

3. **Calculate Target Membership Fraction**
   - Use FnGuide Industry classification
   - Calculate year-specific membership pairs fraction
   - Target: ~2.95% (varies slightly by year)

4. **Calibrate Threshold on Raw Scores**
   - Test thresholds from 0.0 to 0.5 in small increments
   - For each threshold, count pairs where M_raw > threshold
   - Find threshold where fraction ≈ target

5. **Apply Median Adjustment**
   - For each firm i: median_i = median(M_raw[i, :])
   - M_adjusted[i, :] = M_raw[i, :] - median_i

6. **Define TNIC Peer Groups**
   - Use calibrated threshold on M_adjusted
   - Peers_i = {j : M_adjusted[i,j] > threshold}
   - Save peer groups to CSV

## Expected Results

### Network Characteristics

- **Avg peers per firm**: 20-30 (similar to H&P 2016)
- **Peer degree distribution**: Right-skewed (few hubs, many firms with few peers)
- **Network density**: ~2.95% (matching FnGuide Industry)

### Comparison with FnGuide Industry

Three types of peer relationships:
1. **TNIC-only peers**: High text similarity, different FnGuide Industry
2. **FnGuide-only peers**: Same FnGuide Industry, low text similarity
3. **Both**: High text similarity AND same FnGuide Industry

**Hypothesis (H&P 2018):**
- TNIC-only peers capture hidden competitive relationships
- These are less visible to investors
- Should generate stronger momentum effects due to investor inattention
