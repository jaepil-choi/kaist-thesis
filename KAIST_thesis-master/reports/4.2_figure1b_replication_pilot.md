# Phase 4.2: Figure 1B Replication - Conditional Turnover Analysis (Pilot: 2010-2011)

**Generated**: 2025-10-29

## Overview

This document describes the methodology for replicating Figure 1B from Hoberg & Phillips (2018), which analyzes conditional turnover patterns around peer return shocks. Unlike Figure 1A (unconditional analysis), Figure 1B isolates unique effects by examining cases where one peer group experiences high returns while the other peer group has average returns.

### Purpose

- **Replicate**: H&P (2018) Figure 1B "Conditional Turnover around High-Quintile SIC Without TNIC Peer Shock and TNIC Without SIC Peer Shock"
- **Test**: Whether TNIC-only peer shocks generate delayed and persistent turnover patterns compared to FnGuide-only shocks
- **Period**: Pilot implementation using 2010-2011 data (24 months)

### Citation

**Hoberg, G., & Phillips, G. M. (2018).** Text-based industry momentum. *Journal of Financial and Quantitative Analysis*, 53(6), 2355-2388.

### Key Hypothesis

From H&P (2018):
- **FnGuide-only shocks** (visible peers): Immediate turnover response, quick reversion (1-2 months)
- **TNIC-only shocks** (less visible peers): Delayed turnover response starting t+1, persistent elevation (6-12 months)

This pattern supports the **inattention hypothesis**: Investors react slowly to shocks from less visible economic links.

## Methodology

### 1. Data Structures: T×N Wide-Format Matrices

All matrices follow **T×N convention** (time × firms):
- **T** = number of months (24 months for 2010-2011)
- **N** = number of firms with both TNIC and FnGuide data

**Required Matrices:**

| Matrix | Shape | Description |
|--------|-------|-------------|
| `R[t, i]` | T×N | Monthly returns for firm i at time t |
| `turnover[t, i]` | T×N | Monthly turnover for firm i at time t |
| `peer_return_fnguide[t, i]` | T×N | FnGuide-based peer return for firm i at time t |
| `peer_return_tnic[t, i]` | T×N | TNIC-based peer return for firm i at time t |

**Data Sources:**
- **Returns**: Calculated from `data/fnguide/processed/price_wide.parquet`
- **Turnover**: From `data/fnguide/processed/turnover_listed_wide.parquet`
- **FnGuide Industries**: From `data/fnguide/processed/dataguide_groups.parquet` (EOY only)
- **TNIC Peers**: From `data/korean_tnic/by_year/tnic_peers_{year}.csv`
- **TNIC Similarity**: From `data/korean_tnic/by_year/similarity_matrix_{year}.npz`

### 2. Peer Return Calculation: FnGuide Industry

For each firm i at time t, calculate equal-weighted average of peer returns (excluding self):

```python
peer_return_fnguide[t, i] = mean(R[t, peers_fnguide[i]])
```

Where:
- `peers_fnguide[i]` = firms in same FnGuide Industry as firm i (at time t)
- Excludes firm i itself
- Uses time-varying industry membership (EOY classifications applied to full year)

**Implementation Note**: Industry membership is determined by EOY (December) FnGuide classification, applied to all months within that year.

### 3. Peer Return Calculation: TNIC (Matrix Approach)

**Step 1: Create Binary Adjacency Matrix A (N×N)**

```python
# Load median-adjusted similarity matrix
M_adjusted = load_similarity_matrix(year)  # N×N

# Apply threshold to create binary adjacency
threshold = get_calibrated_threshold(year)  # 0.175 for 2010, 0.180 for 2011
A = (M_adjusted > threshold).astype(int)   # N×N

# Set diagonal to zero (exclude self)
np.fill_diagonal(A, 0)
```

**Step 2: Calculate Peer Return Sums**

```python
P = R @ A  # (T×N) @ (N×N) = (T×N)
```

Where:
- `P[t, i]` = sum of returns of all TNIC peers of firm i at time t

**Step 3: Convert to Equal-Weighted Average**

```python
peer_count = A.sum(axis=0)  # (1×N) - number of peers per firm
peer_return_tnic = P / peer_count[np.newaxis, :]  # (T×N)
```

Where:
- `peer_return_tnic[t, i]` = equal-weighted average return of firm i's TNIC peers at time t

**Key Properties:**
- Uses **median-adjusted similarity scores** (asymmetric matrix)
- Uses **calibrated thresholds** from Phase 4.1 TNIC construction
- Peer relationships are firm-specific and non-transitive
- TNIC peers are year-specific (constant within year, change across years)

### 4. Quintile Ranking (Cross-Sectional, Row-by-Row)

For each month t (each row independently):

```python
# Calculate quintile ranks (1-5) for each firm's peer return
quintile_fnguide[t, :] = pd.qcut(peer_return_fnguide[t, :],
                                  q=5, labels=[1,2,3,4,5])

quintile_tnic[t, :] = pd.qcut(peer_return_tnic[t, :],
                               q=5, labels=[1,2,3,4,5])
```

**Quintile Definitions:**
- **Q1**: Bottom 20% (0-20th percentile)
- **Q2**: 20-40th percentile
- **Q3**: **Middle 40-60th percentile** (used as control)
- **Q4**: 60-80th percentile
- **Q5**: **Top 20% (≥80th percentile)** (high peer return shock)

**Output:** Two T×N matrices of quintile ranks
- `quintile_fnguide[t, i]`: Quintile of firm i's FnGuide peer return at time t
- `quintile_tnic[t, i]`: Quintile of firm i's TNIC peer return at time t

### 5. Event Identification: Two Conditional Cases

**Event Case #1: High FnGuide without TNIC**

```python
event_fnguide_only[t, i] = (quintile_fnguide[t, i] == 5) & (quintile_tnic[t, i] == 3)
```

**Interpretation:**
- FnGuide peers experienced high returns (Q5)
- But TNIC peers had average returns (Q3)
- Isolates unique effect of FnGuide peer shock

**Event Case #2: High TNIC without FnGuide**

```python
event_tnic_only[t, i] = (quintile_tnic[t, i] == 5) & (quintile_fnguide[t, i] == 3)
```

**Interpretation:**
- TNIC peers experienced high returns (Q5)
- But FnGuide peers had average returns (Q3)
- Isolates unique effect of TNIC peer shock

**Why Q3 for Control?**
- Q3 represents "average" peer returns (middle of distribution)
- Ensures the other peer group is not experiencing extreme returns
- Allows clean separation of effects

**Output:** Two T×N boolean matrices identifying events
- `event_fnguide_only[t, i]`: Boolean indicator for Event Case #1
- `event_tnic_only[t, i]`: Boolean indicator for Event Case #2

### 6. Event Study: Extract Turnover Windows

For each event identified at (t, i):

**Step 1: Define Window**
- Extract turnover from t-3 to t+12 (16 months total)
- Window: `turnover[t-3:t+13, i]` (Python slicing: t+13 to include t+12)

**Step 2: Data Quality Filters**

Exclude event if any of the following are true:
1. **Missing data**: Any turnover value in window is NaN
2. **Zero turnover**: Any turnover value in window is exactly 0 (indicates suspended trading)
3. **Boundary issues**: t-3 < 0 or t+12 ≥ T (insufficient data before/after)

**Step 3: Overlap Exclusion**

To avoid double-counting:
- If firm i has event at time t, mark (t, t+1, ..., t+15) as "used"
- Subsequent events for firm i within this window are excluded
- Prevents overlapping turnover windows for same firm

**Step 4: Collect Valid Windows**

Separately for each event type:
```python
windows_fnguide_only = []  # List of 16-element arrays
windows_tnic_only = []     # List of 16-element arrays

for t in range(T):
    for i in range(N):
        if event_fnguide_only[t, i] and passes_filters(t, i):
            window = turnover[t-3:t+13, i]
            windows_fnguide_only.append(window)

        if event_tnic_only[t, i] and passes_filters(t, i):
            window = turnover[t-3:t+13, i]
            windows_tnic_only.append(window)
```

### 7. Normalization and Aggregation

**Step 1: Normalize Each Window**

For each window (16 months), normalize by turnover at t=0:

```python
normalized_window = window / window[3]  # Divide by t=0 (index 3 in 0-based array)
```

**Step 2: Average Across All Windows**

Separately for each event type:

```python
avg_turnover_fnguide = np.mean(windows_fnguide_only, axis=0)  # (16,)
avg_turnover_tnic = np.mean(windows_tnic_only, axis=0)        # (16,)
```

**Output:** Two 16-element arrays representing average relative turnover
- `avg_turnover_fnguide[k]`: Average turnover at relative month k-3 for FnGuide-only events
- `avg_turnover_tnic[k]`: Average turnover at relative month k-3 for TNIC-only events

### 8. Visualization: Figure 1B

**Plot Setup:**
- **X-axis**: Relative months from -3 to +12
- **Y-axis**: Relative average stock turnover (normalized to 1.0 at t=0)

**Two Lines:**
1. **"FnGuide Shock w/o TNIC Shock"**: `avg_turnover_fnguide`
2. **"TNIC Shock w/o FnGuide Shock"**: `avg_turnover_tnic`

**Expected Pattern (from H&P 2018 Figure 1B):**

| Event Type | t=0 | t=+1 | t=+2 | t=+12 | Interpretation |
|------------|-----|------|------|-------|----------------|
| **FnGuide-only** | ↑ Spike | Slight decline | Revert to baseline | Baseline | Immediate reaction, quick reversion |
| **TNIC-only** | No spike | ↑ Start increasing | Continue increasing | Elevated | Delayed reaction, persistent elevation |

**Key Finding:**
- **FnGuide shocks**: Immediate turnover response → Quick information incorporation
- **TNIC shocks**: Delayed turnover response → Slow information dissemination (inattention)

### Median Adjustment Application

**Load Median Values:**

From `tnic_metadata_{year}.json`:
```json
{
  "median_adjustment": {
    "mean_median": 0.067327,
    "std_median": 0.017200,
    ...
  }
}
```

**Apply to Similarity Matrix:**

The similarity matrices saved in Phase 4.1 are **raw** (not median-adjusted). We need to apply median adjustment:

```python
# For year 2010
M_raw_2010 = np.load('data/korean_tnic/by_year/similarity_matrix_2010.npz')['similarity']

# Calculate median per firm
medians_2010 = np.median(M_raw_2010, axis=1)  # (N,)

# Apply median adjustment (row-wise)
M_adjusted_2010 = M_raw_2010 - medians_2010[:, np.newaxis]  # (N,N)

# Create binary adjacency matrix
A_2010 = (M_adjusted_2010 > threshold_2010).astype(int)
np.fill_diagonal(A_2010, 0)
```

**Note**: Median adjustment makes the similarity matrix **asymmetric**. This is expected and correct per H&P (2016) methodology.

## Expected Results

### Quantitative Expectations

Based on H&P (2018) Figure 1B results:

**FnGuide-only Events:**
- **N events**: ~500-1,000 (depends on overlap between classifications)
- **t=0**: Turnover spike to ~1.02-1.03 (2-3% increase)
- **t=+2**: Return to baseline ~1.00
- **t=+12**: Stable at baseline ~1.00

**TNIC-only Events:**
- **N events**: ~500-1,000
- **t=0**: No immediate spike, ~0.98-1.00
- **t=+1**: Start increasing to ~1.01
- **t=+3 to t+12**: Persistent elevation ~1.02-1.04 (2-4% above baseline)

### Qualitative Interpretation

**If results match H&P (2018):**
1. **Visibility matters**: Less visible TNIC peers generate delayed responses
2. **Inattention hypothesis supported**: Slow information dissemination for hidden links
3. **Market inefficiency**: TNIC-only shocks create exploitable momentum opportunities

**Korean Market Considerations:**
- May show stronger/weaker effects depending on:
  - Market efficiency
  - Retail vs institutional investor composition
  - Information environment differences from US market
